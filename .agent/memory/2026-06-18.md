# 2026-06-18 - UI endpoints, campos y modo oscuro

## Contexto
El usuario pidio ejecutar `Prompt.md`, con ajustes en `/equipos`, `/campos`,
`/perfil`, `/partidos`, tarjetas clicables en endpoints y el parpadeo blanco del
modo oscuro.

## Cambios
- Cabecera: el enlace autenticado ahora se llama `Perfil`, lleva icono de
  Font Awesome y el boton de cerrar sesion tambien lleva icono.
- Equipos: el boton `Ver mi equipo` para capitanes usa estilo principal
  destacado. La ficha de equipo tiene mas separacion en informacion, invitacion
  y formulario de jugador.
- Campos: se renombro `public/imagenes/jugadores/` a
  `public/imagenes/campos/`. La lista usa esas fotos, completa hasta seis
  tarjetas con catalogo visual si la BD existente tiene menos campos, y el
  sembrado crea seis campos con fotos locales.
- Partidos: el calendario sigue siendo server-side, pero ahora usa estructura
  de calendario real, dias de semana completos, celdas amplias y resumen/hora
  del primer partido del dia.
- Tarjetas clicables: `public/js/principal.js` abre elementos con
  `data-tarjeta-url` si el click no cae sobre controles internos.
- Modo oscuro: `layout.php` aplica `data-tema` antes de cargar CSS usando
  `localStorage` o `prefers-color-scheme`, evitando el flash blanco.
